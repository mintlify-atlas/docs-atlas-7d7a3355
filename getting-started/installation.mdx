`youtubei.js` is published to npm and supports Node.js, Deno, modern browsers, React Native, and Cloudflare Workers. The package ships separate platform entry points so that each environment gets the correct native API shims without unnecessary bundled code.

## Prerequisites

Before installing, make sure your environment meets the following requirements:

<AccordionGroup>
  <Accordion title="Node.js">
    Node.js **18 or later** is required. Node 18 introduced the native `fetch` API and `ReadableStream`, both of which YouTube.js relies on. Earlier versions are not supported.

    Check your version with:
    ```bash
    node --version
    ```
  </Accordion>
  <Accordion title="Deno">
    Any recent stable release of Deno is supported. Deno has built-in TypeScript support and a native `fetch` implementation, so no additional setup is required.
  </Accordion>
  <Accordion title="Browser">
    Any modern browser with support for the Fetch API and `ReadableStream` will work. This covers all current versions of Chrome, Firefox, Safari, and Edge.

    Note that browser environments require a CORS proxy for YouTube API requests. See the platform notes below.
  </Accordion>
</AccordionGroup>

## Install the package

<Tabs>
  <Tab title="npm">
    ```bash
    npm install youtubei.js@latest
    ```
  </Tab>
  <Tab title="Yarn">
    ```bash
    yarn add youtubei.js@latest
    ```
  </Tab>
  <Tab title="pnpm">
    ```bash
    pnpm add youtubei.js@latest
    ```
  </Tab>
  <Tab title="Deno">
    ```bash
    deno add npm:youtubei.js@latest
    ```
  </Tab>
</Tabs>

To install the latest edge version directly from the GitHub repository:

```bash
npm install github:LuanRT/YouTube.js
```

## Package entry points

The package exports multiple entry points. The correct one is resolved automatically in most cases, but you may need to specify it explicitly in certain build environments.

| Entry point | Use case |
|---|---|
| `youtubei.js` | Node.js (resolved automatically via package exports) |
| `youtubei.js/web` | Explicit browser import or bundler target |
| `youtubei.js/react-native` | React Native projects |
| `youtubei.js/cf-worker` | Cloudflare Workers |
| `youtubei.js/agnostic` | Platform-agnostic core with no shims (advanced use) |

In Node.js, Deno, and standard browser bundler setups, a plain `import { Innertube } from 'youtubei.js'` will resolve to the correct entry point.

## Platform-specific notes

<AccordionGroup>
  <Accordion title="Node.js">
    The default import resolves to the Node.js entry point automatically. No additional configuration is required.

    ```typescript
    import { Innertube } from 'youtubei.js';

    const yt = await Innertube.create();
    ```

    <Note>
      If you are using CommonJS (`require`), the package is ESM-only. Use a dynamic `import()` or migrate to ES modules.
    </Note>
  </Accordion>
  <Accordion title="Deno">
    After running `deno add npm:youtubei.js@latest`, import as you would in Node.js:

    ```typescript
    import { Innertube } from 'youtubei.js';

    const yt = await Innertube.create();
    ```

    The Deno entry point is resolved automatically via the package's `exports` field.
  </Accordion>
  <Accordion title="Browser">
    In a browser environment, YouTube API requests will be blocked by CORS unless you route them through a proxy. You pass your own `fetch` implementation to `Innertube.create()` that rewrites requests to go through your proxy server.

    ```typescript
    import { Innertube } from 'youtubei.js/web';

    function proxiedFetch(input: RequestInfo | URL, init?: RequestInit) {
      const url = new URL(typeof input === 'string' ? input : input instanceof URL ? input : input.url);
      // Rewrite the host to your proxy
      url.searchParams.set('__host', url.host);
      url.host = 'your-proxy.example.com';
      return fetch(url.toString(), init);
    }

    const yt = await Innertube.create({
      fetch: proxiedFetch,
      generate_session_locally: true,
    });
    ```

    <Warning>
      Never expose your YouTube session cookies or credentials in client-side browser code.
    </Warning>
  </Accordion>
  <Accordion title="React Native">
    Import from the `react-native` entry point, which bundles the correct shims for React Native's JS environment:

    ```typescript
    import { Innertube } from 'youtubei.js/react-native';

    const yt = await Innertube.create();
    ```

    The pre-bundled file at `bundle/react-native.js` is also available if you are using Metro or another bundler that does not support package `exports`.
  </Accordion>
  <Accordion title="Cloudflare Workers">
    Import from the `cf-worker` entry point:

    ```typescript
    import { Innertube } from 'youtubei.js/cf-worker';

    export default {
      async fetch(request: Request): Promise<Response> {
        const yt = await Innertube.create();
        // ...
        return new Response('ok');
      }
    };
    ```

    <Note>
      Cloudflare Workers have restrictions on dynamic code evaluation. The `generate_session_locally: true` option is recommended to avoid issues with player script evaluation.
    </Note>
  </Accordion>
</AccordionGroup>

## Verify the installation

Once installed, confirm everything is working with this minimal script:

```typescript
import { Innertube } from 'youtubei.js';

const yt = await Innertube.create();
console.log('Session created. Client version:', yt.session.client_version);
```

If you see a client version string printed without errors, your installation is working correctly.
