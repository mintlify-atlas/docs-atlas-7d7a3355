The `Innertube` class is the primary interface for YouTube.js. It wraps an authenticated `Session` and exposes every major InnerTube API surface — from fetching video metadata and streaming data to browsing feeds, managing comments, and delegating to specialized clients like YouTube Music and YouTube Studio.

## Static methods

### `create`

```typescript
static async create(config?: InnerTubeConfig): Promise<Innertube>
```

Creates a new `Innertube` instance by initializing a `Session` with the provided configuration.

```typescript
import { Innertube, UniversalCache } from 'youtubei.js';

const yt = await Innertube.create({ cache: new UniversalCache(true) });
```

<ParamField path="config" type="InnerTubeConfig">
  Optional session configuration. All fields are optional. See [InnerTubeConfig](#innertube-config) below.
</ParamField>

**Returns** `Promise<Innertube>`

## InnerTubeConfig

`InnerTubeConfig` is an alias for `SessionOptions`. All fields are optional.

<ParamField path="lang" type="string">
  BCP 47 language tag for the session (e.g. `"en"`, `"pt-BR"`). Defaults to the server's detected language.
</ParamField>

<ParamField path="location" type="string">
  ISO 3166-1 alpha-2 country code for geolocation (e.g. `"US"`, `"GB"`).
</ParamField>

<ParamField path="user_agent" type="string">
  Custom user-agent string used in InnerTube requests.
</ParamField>

<ParamField path="account_index" type="number">
  Zero-based index of the account to use when multiple accounts are signed in via cookies.
</ParamField>

<ParamField path="on_behalf_of_user" type="string">
  Page ID of the YouTube profile/channel to act as when the account has multiple profiles.
</ParamField>

<ParamField path="retrieve_player" type="boolean" default="true">
  Whether to fetch the JS player during session creation. Disable to speed up initialization, but note that format deciphering will not work without it.
</ParamField>

<ParamField path="enable_safety_mode" type="boolean" default="false">
  Prevent the session from loading potentially unsafe content.
</ParamField>

<ParamField path="retrieve_innertube_config" type="boolean" default="true">
  Whether to retrieve the InnerTube config data. Useful for onesie requests.
</ParamField>

<ParamField path="generate_session_locally" type="boolean" default="false">
  Generate session data locally instead of fetching it from YouTube. Faster, but may be less accurate.
</ParamField>

<ParamField path="fail_fast" type="boolean" default="false">
  When `true`, session creation throws if YouTube session data cannot be retrieved instead of falling back to a local session.
</ParamField>

<ParamField path="enable_session_cache" type="boolean" default="true">
  Whether to cache session data using the provided `cache` implementation.
</ParamField>

<ParamField path="device_category" type="string">
  Platform category for the session. Accepted values: `"desktop"`, `"mobile"`, `"tv"`.
</ParamField>

<ParamField path="client_type" type="ClientType">
  InnerTube client type. See [`ClientType`](/api-reference/session#clienttype-enum) for all values.
</ParamField>

<ParamField path="timezone" type="string">
  IANA timezone string (e.g. `"America/New_York"`). Defaults to the system timezone.
</ParamField>

<ParamField path="cache" type="ICache">
  Cache implementation for algorithms, session data, and OAuth2 tokens. Pass `new UniversalCache(true)` for persistent file-system caching.
</ParamField>

<ParamField path="cookie" type="string">
  Raw YouTube cookie string for cookie-based authentication.
</ParamField>

<ParamField path="visitor_data" type="string">
  Persistent visitor data string. Lets YouTube serve tailored content even without authentication.
</ParamField>

<ParamField path="fetch" type="FetchFunction">
  Custom `fetch`-compatible function to use for all HTTP requests.
</ParamField>

<ParamField path="po_token" type="string">
  Session-bound Proof of Origin Token (PoT). An attestation token generated by BotGuard/DroidGuard used to verify the request originates from a real client.
</ParamField>

<ParamField path="player_id" type="string">
  Override the JS player ID. Useful to force an older player when YouTube introduces breaking changes.
</ParamField>

## Video methods

### `getInfo`

```typescript
async getInfo(target: string | NavigationEndpoint, options?: GetVideoInfoOptions): Promise<VideoInfo>
```

Fetches full video info, including watch-next recommendations and playlist context, by calling both the `watch` and `watchNext` endpoints in parallel.

Use this when you need sidebar recommendations, related videos, or playlist context in addition to the video metadata. For metadata alone, prefer [`getBasicInfo`](#getbasicinfo).

```typescript
const info = await yt.getInfo('dQw4w9WgXcQ');
console.log(info.basic_info.title);
```

<ParamField path="target" type="string | NavigationEndpoint" required>
  Either a plain video ID string or a `NavigationEndpoint` instance (e.g. from a search result or playlist item).
</ParamField>

<ParamField path="options" type="GetVideoInfoOptions">
  <Expandable title="properties">
    <ParamField path="client" type="InnerTubeClient">
      InnerTube client to use for this request. Accepted values: `"IOS"`, `"WEB"`, `"MWEB"`, `"ANDROID"`, `"ANDROID_VR"`, `"YTMUSIC"`, `"YTMUSIC_ANDROID"`, `"YTSTUDIO_ANDROID"`, `"TV"`, `"TV_SIMPLY"`, `"TV_EMBEDDED"`, `"YTKIDS"`, `"WEB_EMBEDDED"`, `"WEB_CREATOR"`.
    </ParamField>
    <ParamField path="po_token" type="string">
      Proof of Origin Token bound to this specific video ID. Falls back to the session-level `po_token` if not provided.
    </ParamField>
  </Expandable>
</ParamField>

**Returns** `Promise<VideoInfo>`

---

### `getBasicInfo`

```typescript
async getBasicInfo(video_id: string, options?: GetVideoInfoOptions): Promise<VideoInfo>
```

Fetches video metadata using only the `watch` endpoint — no watch-next data. Faster than `getInfo` and sufficient when you only need video metadata, formats, and captions.

```typescript

const format = info.chooseFormat({ type: 'video+audio', quality: '360p' });
```

  The YouTube video ID.
</ParamField>

<ParamField path="options" type="GetVideoInfoOptions">
  <Expandable title="properties">
    <ParamField path="client" type="InnerTubeClient">
      InnerTube client to use for this request.
    </ParamField>
    <ParamField path="po_token" type="string">
      Video-bound Proof of Origin Token.
    </ParamField>
  </Expandable>
</ParamField>

### `getStreamingData`

```typescript
async getStreamingData(video_id: string, options?: FormatOptions): Promise<Format>
```

Returns a single deciphered streaming `Format` object with a ready-to-use download URL. An alternative to `download` when you only need the direct URL without streaming through the library.

```typescript
const format = await yt.getStreamingData('dQw4w9WgXcQ', {
  type: 'video+audio',
  quality: '720p',
  format: 'mp4'
});
console.log(format.url);
```

<ParamField path="video_id" type="string" required>
  The YouTube video ID.
</ParamField>

<ParamField path="options" type="FormatOptions">
  <Expandable title="properties">
    <ParamField path="itag" type="number">
      Specific video or audio itag to select.
    </ParamField>
    <ParamField path="quality" type="string">
      Video quality label: `"360p"`, `"720p"`, `"1080p"`, `"best"`, `"bestefficiency"`, etc.
    </ParamField>
    <ParamField path="type" type="string">
      Download type: `"video"`, `"audio"`, or `"video+audio"`.
    </ParamField>
    <ParamField path="language" type="string" default="original">
      Language code for audio track selection.
    </ParamField>
    <ParamField path="format" type="string">
      Container format (e.g. `"mp4"`, `"webm"`). Use `"any"` to disable filtering.
    </ParamField>
    <ParamField path="codec" type="string">
      Codec preference, e.g. `"avc"`, `"vp9"`, `"av01"` for video; `"opus"`, `"mp4a"` for audio.
    </ParamField>
    <ParamField path="client" type="InnerTubeClient">
      InnerTube client override.
    </ParamField>
    <ParamField path="po_token" type="string">
      Video-bound Proof of Origin Token.
    </ParamField>
  </Expandable>
</ParamField>

**Returns** `Promise<Format>`

### `download`

```typescript
async download(video_id: string, options?: DownloadOptions): Promise<ReadableStream<Uint8Array>>
```

Downloads a video and returns a `ReadableStream`. Internally calls `getBasicInfo` and then `VideoInfo.download`.

```typescript
import { createWriteStream } from 'node:fs';
import { Readable } from 'node:stream';

const stream = await yt.download('dQw4w9WgXcQ', {
  type: 'video+audio',
  quality: 'best',
  format: 'mp4'
});

Readable.fromWeb(stream).pipe(createWriteStream('video.mp4'));
```

<ParamField path="video_id" type="string" required>
  The YouTube video ID.
</ParamField>

<ParamField path="options" type="DownloadOptions">
  <Expandable title="properties">
    <ParamField path="itag" type="number">
      Specific itag to download.
    </ParamField>
    <ParamField path="quality" type="string">
      Video quality label.
    </ParamField>
    <ParamField path="type" type="string">
      `"video"`, `"audio"`, or `"video+audio"`.
    </ParamField>
    <ParamField path="language" type="string" default="original">
      Audio track language code.
    </ParamField>
    <ParamField path="format" type="string">
      Container format.
    </ParamField>
    <ParamField path="codec" type="string">
      Codec preference.
    </ParamField>
    <ParamField path="range" type="object">
      Byte range to download.
      <Expandable title="properties">
        <ParamField path="start" type="number" required>
          Start byte (inclusive).
        </ParamField>
        <ParamField path="end" type="number" required>
          End byte (inclusive).
        </ParamField>
      </Expandable>
    </ParamField>
    <ParamField path="client" type="InnerTubeClient">
      InnerTube client override.
    </ParamField>
    <ParamField path="po_token" type="string">
      Video-bound Proof of Origin Token.
    </ParamField>
  </Expandable>
</ParamField>

**Returns** `Promise<ReadableStream<Uint8Array>>`

## Search methods

### `search`

```typescript
async search(query: string, filters?: SearchFilters): Promise<Search>
```

Searches YouTube with optional filters.

```typescript
const results = await yt.search('lo-fi music', {
  type: 'video',
  upload_date: 'week',
  features: ['hd']
});
```

<ParamField path="query" type="string" required>
  The search query string.
</ParamField>

<ParamField path="filters" type="SearchFilters">
  <Expandable title="properties">
    <ParamField path="upload_date" type="string">
      Filter by upload date. One of: `"all"`, `"today"`, `"week"`, `"month"`, `"year"`.
    </ParamField>
    <ParamField path="type" type="string">
      Result type filter. One of: `"all"`, `"video"`, `"shorts"`, `"channel"`, `"playlist"`, `"movie"`.
    </ParamField>
    <ParamField path="duration" type="string">
      Duration filter. One of: `"all"`, `"under_three_mins"`, `"three_to_twenty_mins"`, `"over_twenty_mins"`.
    </ParamField>
    <ParamField path="prioritize" type="string">
      Result ranking. One of: `"relevance"`, `"popularity"`.
    </ParamField>
    <ParamField path="features" type="string[]">
      Content features to filter by. Accepted values: `"hd"`, `"subtitles"`, `"creative_commons"`, `"3d"`, `"live"`, `"purchased"`, `"4k"`, `"360"`, `"location"`, `"hdr"`, `"vr180"`.
    </ParamField>
  </Expandable>
</ParamField>

**Returns** `Promise<Search>`

## Utility methods

### `resolveURL`

```typescript
async resolveURL(url: string): Promise<NavigationEndpoint>
```

Resolves a YouTube URL to a `NavigationEndpoint`, which you can then pass to methods like `getInfo` or `call`.

```typescript
const endpoint = await yt.resolveURL('https://www.youtube.com/watch?v=dQw4w9WgXcQ');
const info = await yt.getInfo(endpoint);
```

<ParamField path="url" type="string" required>
  A full YouTube URL to resolve.
</ParamField>

**Returns** `Promise<NavigationEndpoint>`

### `call`

```typescript
call<T extends IParsedResponse>(endpoint: NavigationEndpoint, args: { [key: string]: any; parse: true }): Promise<T>
call(endpoint: NavigationEndpoint, args?: { [key: string]: any; parse?: false }): Promise<ApiResponse>
```

A utility method to call any `NavigationEndpoint` directly without using `Actions`. Pass `parse: true` to receive a parsed response object.

```typescript
const endpoint = new NavigationEndpoint({ browseEndpoint: { browseId: 'FEwhat_to_watch' } });
const response = await yt.call(endpoint, { parse: true });
```

<ParamField path="endpoint" type="NavigationEndpoint" required>
  The navigation endpoint to invoke.
</ParamField>

<ParamField path="args" type="object">
  Additional payload fields to merge into the request. Set `parse: true` to return a parsed `IParsedResponse` instead of a raw `ApiResponse`.
</ParamField>

**Returns** `Promise<IParsedResponse | ApiResponse>`
